## Что это такое

GDEX — веб-приложение для скачивания уровней Geometry Dash с серверов RobTop и их экспорта в форматы, удобные вне игры: GMD для повторного импорта, JSON для анализа, CSV/XLS для таблиц, кастомная музыка уровня в MP3.

Аудитория: создатели уровней, которым нужен бэкап своих работ, исследователи дизайна уровней GD и те, кто собирает датасеты размещения объектов для ML.

---

## Что делает

1. Ты вводишь ID уровня.
2. GDEX запрашивает у GDBrowser метаданные (название, автор, сложность, звёзды, скачивания).
3. Ты выбираешь нужные форматы.
4. Жмёшь Download. Сервер забирает сырые данные уровня с Boomlings (или фейловером с GD Proxy), распаковывает их, парсит поток объектов и отдаёт структурированный JSON.
5. Браузер собирает запрошенные файлы из этого JSON и предлагает их к скачиванию.

Для нескольких уровней вкладка Batch принимает список (или диапазон вида `128-200`), запускает тот же поток для каждого и показывает прогресс по каждому элементу. Упавшие элементы можно повторить по одному.

## Форматы вывода

### GMD

XML plist в формате внутриигрового `.gmd`. Бросаешь в диалог Import редактора Geometry Dash — уровень появляется локально. Содержит полную декодированную строку уровня, не превью.

### JSON

Структурированный снапшот с метаданными, заголовком уровня, первыми 100 объектами, отсортированными по X, и агрегатной статистикой (количество объектов, размеры, топ-типы). Подходит для sequence-моделей и анализа в целом.

### CSV / XLS

Объектный дамп: каждая строка — объект, столбцы — все его свойства. CSV открывается в любом табличном редакторе. XLS — это TSV с MIME-типом Excel, поэтому Excel открывает его сразу, без диалога импорта.

### Metadata

Только блок информации об уровне в JSON: автор, сложность, песня, даты загрузки и обновления, количество монет, флаги верификации. Без данных по объектам.

### MP3

Кастомная песня уровня, отдаётся напрямую с её оригинального URL на Newgrounds. Молча пропускается, если песня заблокирована по региону или больше не хостится.

> Данные по объектам ограничены первыми 100 на уровень для экспортов JSON, CSV и XLS. Полная декодированная строка уровня попадает только в GMD.

## Установка и запуск

```sh
bun install
bun dev
```

Dev-сервер крутится на `http://localhost:3000` через Turbopack.

**Требования:** Bun ≥ 1.0 или Node ≥ 20.9 с npm.

---

## Продакшн

```sh
bun build
bun start
```

Билд создаёт каталог `.next/standalone/`, готовый к деплою на любой Node-хост. В `next.config.ts` указан `output: "standalone"`, поэтому зависимости вшиваются в рантайм-образ.

### Vercel

Пушнул в GitHub, импортнул репу на Vercel. Фреймворк-пресет определится как Next.js автоматически. Дополнительной конфигурации не требуется.

### Self-host

```sh
bun run build
node .next/standalone/server.js
```

Слушает на `PORT` (по дефолту `3000`).

## Конфигурация

У GDEX нет обязательных переменных окружения. Все дефолты вшиты в код.

GD-секрет, которым клиент авторизуется на Boomlings — публично известная константа, одна и та же во всех релизах Geometry Dash. Лежит в [`lib/constants.ts`](./lib/constants.ts) как `GD_CLIENT_SECRET`, не в env.

Что можно подкрутить в коде:

| Константа | Дефолт | Файл |
|---|---|---|
| `MAX_LEVEL_ID_LENGTH` | `12` | `app/api/check/route.ts`, `app/api/download/route.ts` |
| `MAX_RAW_BYTES` | `10485760` (10 МБ) | `app/api/download/route.ts` |
| `MAX_DECODED_BYTES` | `67108864` (64 МБ) | `app/api/download/route.ts` |
| Rate-лимиты | `30/мин`, `12/мин`, `10/мин` | `app/api/check/route.ts`, `app/api/servers/route.ts`, `app/api/download/route.ts` |

## Что приложение трогает

> [!IMPORTANT]
>
> | | |
> |---|---|
> | Внешние сервисы | `gdbrowser.com`, `boomlings.com`, `gdproxy.net` |
> | Диск сервера | Ничего. Нет БД, нет файлового хранилища |
> | RAM сервера | Бакеты rate-лимита в памяти, временные буферы данных уровня на время запроса |
> | Browser localStorage | Тема (`theme: "dark" \| "light"`) |
> | Исходящая сеть | HTTPS на три сервера Geometry Dash выше |

Нет аккаунтов и нет учётных данных. «Секрет» Boomlings — это публичная константа GD-клиента `Wmfd2893gb7`, идентичная во всех релизах игры.

Серверное состояние — попроцессное и эфемерное. Перезапустишь процесс — счётчики rate-лимита сбросятся.

Чтобы сбросить состояние клиента:
- Переключи тему через кнопку в шапке
- Очисти данные сайта в браузере

## Как это работает

Запрос из браузера на `/api/download` приходит в Node-runtime роут. Хендлер валидирует тело, ищет выбранный сервер в `GD_SERVERS` и пытается обратиться к нему первым. Если сервер вернул осмысленный ответ (длиннее 10 символов и не начинается с `-1`), хендлер идёт дальше. Иначе фейловерится на следующий сервер из списка.

Сырой ответ — это base64 с deflate внутри. `decodeLevelData` пробует несколько вариантов zlib `windowBits` подряд через `zlib.inflateSync`, с лимитом выхода 64 МБ — защита от декомпрессионных бомб. Декодированный текст — поток, разделённый `;`, где первый сегмент — заголовок уровня, остальные — записи объектов (пары ключ-значение через `,`).

Хендлер возвращает декодированную строку уровня, превью разобранных объектов, заголовок, посчитанную статистику и блок информации о песне, если он есть. Экспортеры в `lib/exporters.ts` собирают конкретные файлы для скачивания на стороне браузера: GMD оборачивает строку уровня в plist, CSV и XLS итерируют по разобранным объектам, JSON ресериализует всё одним структурированным payload-ом.

Rate-лимит — это in-memory мапа token-bucket по ключу `x-forwarded-for` (или `x-real-ip`, или `"anonymous"`). Только в пределах одного процесса. См. Заметки.

Тема избегает мерцания неправильной темы за счёт inline-скрипта в `<head>`, который читает `localStorage["theme"]` (или `prefers-color-scheme`) и ставит `data-theme` до отрисовки. `ThemeProvider` потом читает этот атрибут как стартовое состояние, поэтому сервер и клиент договариваются на первом рендере.

## Заметки и ограничения

**Rate-лимит in-memory и попроцессный.** Норм для single-instance деплоя (дефолтный Vercel, одна VM, один контейнер). На горизонтально масштабированном деплое каждая реплика держит свой лимит, и эффективный потолок становится `реплики × лимит`. Перед скейлом замени `lib/rateLimit.ts` на бэкенд через Redis или Vercel KV.

**GD-секрет — это не секрет.** `Wmfd2893gb7` — то же значение, зашитое в каждый релиз Geometry Dash. Хранится как константа, а не env-var, чтобы это было явно. Прятать его в env означало бы намекать на security-свойство, которого нет.

**Декомпрессия ограничена 64 МБ.** `zlib.inflateSync` запускается с `maxOutputLength` для защиты от подстроенных ответов сверху. Реальные уровни GD сильно меньше. Если запрос упрётся в лимит — роут вернёт 413.

**Экспорт MP3 кастомной песни зависит от Newgrounds.** Ссылка тянется напрямую из блока инфы о песне уровня. Песни с региональными ограничениями или гео-блоками вернут 403 в браузере пользователя, и серверного прокси для обхода тут нет.

**Нет аккаунтов, нет авторизации.** GDEX полностью анонимен. Не во что заходить, нечего отзывать, ничего не хранится per-user кроме темы в localStorage.

**Билд требует доступа в интернет к Google Fonts.** `next/font/google` тянет Geist и Geist Mono на этапе сборки. CI без интернета упадёт. Чтобы билд был самодостаточным, переключись на `next/font/local` и положи WOFF2 в `public/fonts/`.

**Стек:** Next.js 16, React 19, TypeScript 5 strict, Tailwind CSS v4, Bun. Без БД, без ORM, без state-management библиотеки, без UI-кит библиотеки.
